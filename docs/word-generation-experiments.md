## 🔬 실험 1: 카테고리별 전문 용어 집중 생성

### 가설
카테고리별로 실제 SaaS 서비스에서 많이 사용하는 전문 용어를 집중 생성하면 품질이 높을 것이다.

### 방법
각 카테고리별로 실제 업계 용어, 기술 용어, 비즈니스 용어를 최대한 많이 생성

### 결과

#### 실험 1-1: Analytics 카테고리
| 항목 | 결과 |
|------|------|
| 총 단어 | 1,403개 |
| **통과** | **1,370개 (97.6%)** ✅ |
| 실패 | 33개 |
| **평균 점수** | **99.6점** ✅ |
| 실행 시간 | 4ms |

#### 실험 1-2: Growth 카테고리  
| 항목 | 결과 |
|------|------|
| 총 단어 | 4,093개 |
| **통과** | **4,039개 (98.7%)** ✅ |
| 실패 | 54개 |
| **평균 점수** | **99.8점** ✅ |
| 실행 시간 | 6ms |

### ✅ 발견한 노하우

#### 1. 카테고리별 집중 생성이 매우 효과적이다
- **통과율**: 97.6% ~ 98.7% (매우 높음)
- **품질**: 99.6 ~ 99.8점 (거의 완벽)
- **이유**: 실제 업계에서 사용하는 전문 용어라 SaaS 적합성이 높음

#### 2. 한 번에 대량 생성이 가능하다
- **Analytics**: 1,400개 단어 생성
- **Growth**: 4,000개 단어 생성
- **속도**: 4~6ms (매우 빠름)

#### 3. 전문 용어 패턴을 활용하면 품질이 높다
**실제 업계 패턴**:
```
기본 용어 + 변형 → 새 용어
- analytics → data analytics, predictive analytics
- growth → growth hacking, viral growth
- metrics → KPI metrics, leading indicators
```

#### 4. 주요 실패 원인 파악
- **짧은 단어**: 85% 실패 (2자 이하: io, ve, ai, ml 등)
- **연속된 하이픈**: --double, -prefix
- **영어 아님**: 한글, 일본어, 특수문자 포함

#### 5. 성공적인 단어 패턴
```
✅ 복합 명사: workflow, workspace, marketplace
✅ 동사+명사: forecasting, reporting, analyzing
✅ 기술 용어: autocorrelation, stationarity, cointegration
✅ 비즈니스 용어: ebitda, mrr, arr, ltv, cac
✅ 약어 형태: kpi, okr, roi, ceo, cto
✅ 하이픈 연결: data-driven, evidence-based, cost-effective
```

## 🔬 실험 2: 접미사 조합 패턴

### 가설
기존 단어에 접미사를 붙이면 대량으로 확장 가능할 것이다.

### 방법
analytics/growth 카테고리 단어에 접미사 추가 (-pro, -plus, -cloud, -io, -svc, -api, -app, -hq, -lab, -hub, -sys, -net, -ware, -ify)

### 결과
| 항목 | 결과 |
|------|------|
| 총 단어 | 4,897개 |
| **통과** | **4,388개 (89.6%)** ⚠️ |
| 실패 | 509개 |
| **평균 점수** | **98.0점** ✅ |
| 실행 시간 | 10ms |

### ⚠️ 발견한 문제점

#### 1. 접미사 조합은 길이 제한 위반이 많다
- **실패 원인 1위**: "너무 깁니다 (최대 20자)" - 467건 (91.7%)
- **이유**: 기존 단어(10-15자) + 접미사(3-4자) = 20자 초과
- **예시**: `autocorrelation-pro` (20자 초과), `stationarity-plus` (16자 - 통과)

#### 2. 통과율이 카테고리별 집중 생성보다 낮다
- **접미사 패턴**: 89.6% 통과
- **카테고리별 집중**: 97.6% ~ 98.7% 통과
- **차이**: 약 8-9% 낮음

#### 3. 품질은 여전히 높다
- 평균 점수: 98.0점 (높은 편)
- 통과한 단어들은 실제 SaaS 제품명으로 사용 가능

### ✅ 개선된 노하우

#### 1. 접미사 추가 시 길이 사전 체크 필요
```
접미사 추가 전 길이 체크
if (기존 단어 길이 + 접미사 길이 > 20자) {
    건너뛰기
}
```

#### 2. 짧은 기본 단어에만 접미사 추가
- **권장**: 10자 이하 기본 단어 + 접미사
- **예**: `analytics` (9자) + `-pro` (4자) = `analytics-pro` (13자) ✅

#### 3. 짧은 접미사 우선 사용
- **추천 접미사**: -io, -app, -hq, -lab, -fy (3자 이하)
- **비추천**: -service, -platform, -solution (너무 김)

- 이는 모든 실험 중 최고의 결과

#### 2. 길이 제한의 중요성 확인
- 실험 2 (무제한 접미사): 89.6% 통과
- 실험 3 (길이 제한): 100% 통과
- **향상폭**: +10.4%

#### 3. 짧은 기본 단어가 핵심
- 사용된 기본 단어: analytics(9), metrics(7), insights(8), data(4), cloud(5) 등

### ✅ 확립된 노하우

```
기본 단어 선택: 10자 이하

- 짧은 단어라도 20자 초과 위험

## 🔬 실험 4: 숫자 + 단어 조합

### 가설

### 🎯 또 다른 완벽한 성공!

#### 1. 숫자 조합도 완벽한 통과율
- **100% 통과**: 실패 단어 0개
- **평균 점수**: 만점 100.0점
- 실험 3에 이어 두 번째 완벽한 결과

#### 2. 숫자 패턴의 장점
- **현대적 느낌**: analytics2, metrics365
- **의미 부여 가능**: 24(24시간), 365(1년), 360(전체)

- **10**: 완벽/상위 (예: top10)
- **360**: 전체/포괄 (예: view360)
- **99**: 거의 완벽 (예: sync99)

### ✅ 확립된 노하우

#### 1. 완벽한 숫자 패턴
```
기본 단어 + 숫자 (붙여쓰기 또는 하이픈)

- **버전형**: app1, web2, tech3
- **시간형**: data24, sync24, flow-24
- **연간형**: metrics365, analytics-365
- **완전형**: view360, insights-360
- **상위형**: top10, best-10

#### 3. 하이픈 사용 여부
- 붙여쓰기: analytics2 (더 현대적)
- 하이픈: analytics-2 (더 가독성)
- 둘 다 QA 통과
